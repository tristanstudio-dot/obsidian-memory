# Obsidian Links - Принцип построения связей

**Дата:** 20 апреля 2026

## 🔗 ОСНОВНОЙ ПРИНЦИП

Связи в Obsidian создаются через **вики-ссылки** [[]].

**Синтаксис:**
\\\
[[Projects/HAYDA]]
\\\

Когда ты пишешь такую ссылку:
1. Obsidian ищет файл с именем **HAYDA.md** в папке **Projects**
2. Создаёт связь между текущим файлом и файлом HAYDA.md
3. В графе появляется линия между двумя узлами
4. Если кликнуть на ссылку → откроется файл HAYDA.md

---

## 📊 ТИПЫ СВЯЗЕЙ

### 1. ДВУСТОРОННЯЯ СВЯЗЬ (Backlinks)

**Если в файле A пишешь:**
\\\
[[Projects/HAYDA]]
\\\

**Obsidian автоматически показывает:**
- В файле HAYDA.md внизу появляется **Backlinks** раздел
- Там видно что на HAYDA.md ссылаются из файла A
- Это **двусторонняя связь** (A → B AND B ← A)

**В графе:**
- Видна линия между файлом A и HAYDA.md
- Связь ВИДНА в обе стороны

### 2. ОДНОСТОРОННЯЯ СВЯЗЬ

**Если только в файле A пишешь ссылку на B:**
\\\
Смотри [[Projects/HAYDA]] для деталей.
\\\

**Результат:**
- Файл A → Файл B
- Если открыть файл B, в Backlinks увидишь ссылку на A
- **В графе:** Линия видна в обе стороны (потому что Obsidian показывает обе стороны автоматически)

### 3. ССЫЛКА НА ЗАГОЛОВОК (Header Link)

\\\
[[Projects/HAYDA#Три пакета]]
\\\

**Результат:**
- Ссылка на конкретный заголовок внутри файла
- При клике откроется файл и прокрутится до этого заголовка
- Создаёт связь между файлами

### 4. ССЫЛКА С КРАСИВЫМ ТЕКСТОМ

\\\
[[Projects/HAYDA|Главный проект]]
\\\

**Результат:**
- Вместо \[[Projects/HAYDA]]\ видно \Главный проект\
- Но ссылка всё ещё указывает на HAYDA.md
- **В графе:** связь создана нормально, текст только для читаемости

## 🎯 ПРАВИЛА ХОРОШИХ СВЯЗЕЙ

### ✅ ДЕЛАЙ ТАК

1. **Линкуй только релевантные файлы**
   - ✅ \[[Projects/HAYDA]]\ - точная ссылка
   - ✅ \[[Daily_Log/Hayda_Log_2026-04-20]]\ - связана с днём
   - ❌ \[[References/Random_File]]\ - если не относится к HAYDA

2. **Используй иерархию папок в ссылках**
   - ✅ \[[Projects/HAYDA]]\ - полный путь
   - ✅ \[[Daily_Log/2026-04]]\ - по месяцам
   - ❌ \[[HAYDA]]\ - если файлов с таким именем несколько

3. **Добавляй ссылки в начало/конец файла**
   - Вверху: **Связанные документы:**
   - Внизу: **Смотри также:**
   - Внутри: \[[Projects/HAYDA#Раздел]]\ для раздела

4. **Используй разные ссылки для разных целей**
   - На проект: \[[Projects/HAYDA]]\
   - На версию: \[[Projects/HAYDA|v1.1]]\
   - На раздел: \[[Projects/HAYDA#Finance]]\

### ❌ ИЗБЕГАЙ ЭТОГО

1. **Циклические ссылки (если не нужны)**
   - ❌ A → B → C → A (бесконечный цикл)
   - ✅ OK если это логично (например обратная ссылка)

2. **Слишком много связей от одного файла**
   - ❌ Файл ссылается на 50 других (спагетти)
   - ✅ Ссылайся только на релевантные

3. **Broken links (ссылки на несуществующие файлы)**
   - ❌ \[[Файл_которого_нет]]\ - будет серая ссылка
   - ✅ Проверяй что файл существует перед ссылкой

4. **Неправильная иерархия**
   - ❌ \[[HAYDA]]\ если нет HAYDA.md в корне
   - ✅ \[[Projects/HAYDA]]\ правильный путь

## 🔍 КАК OBSIDIAN ОТСЛЕЖИВАЕТ СВЯЗИ

### Внутренне Obsidian делает ТАК:

1. **Сканирует все файлы** при открытии Obsidian
2. **Парсит каждый файл** ищет вики-ссылки \[[]]\
3. **Создаёт граф-базу данных** где каждая ссылка = рёбра графа
4. **Обновляет в реальном времени** когда ты редактируешь

**Пример:**
\\\
Projects/HAYDA.md содержит: [[Projects/Splendid_Idea]]
                                    ↓
Projects/Splendid_Idea.md содержит: [[Projects/HAYDA]]
                                    ↓
Obsidian создаёт ДВУСТОРОННИЕ рёбра между ними:
  HAYDA.md ←→ Splendid_Idea.md
\\\

### Backlinks (обратные ссылки)

Когда ты открываешь файл, внизу видишь **Backlinks** раздел.

Это ВСЕ файлы которые ссылаются НА текущий файл.

**Пример:**
- Открыл Projects/HAYDA.md
- Внизу вижу Backlinks:
  - Daily_Log/Hayda_Log_2026-04-20.md (ссылается на HAYDA)
  - Projects/Splendid_Idea.md (ссылается на HAYDA)
  - References/Graph_Guide.md (ссылается на HAYDA)

## 📊 ВИЗУАЛЬНО КАК ЭТО ВЫГЛЯДИТ

### В графе это выглядит так:

\\\
        Daily_Log
           ↓
           HAYDA (центр)
         ↗     ↖
  Splendid    Graph_Guide
\\\

Каждая стрелка = вики-ссылка в коде

### В файле это выглядит так:

\\\markdown
# HAYDA

Связанные проекты:
- [[Projects/Splendid_Idea]] ← ссылка
- [[Projects/HIDO_Poexali]] ← ссылка

Когда откроешь этот файл, в Backlinks внизу:
- Файлы которые ссылаются на этот
- Все обратные ссылки
\\\

## 🔑 КЛЮЧЕВЫЕ МОМЕНТЫ

1. **Связи создаются автоматически** - просто пиши \[[]]\
2. **Двусторонние** - Obsidian автоматически показывает обе стороны
3. **Видны в Backlinks** - открыи файл, внизу видишь кто ссылается
4. **Видны в графе** - как линии между узлами
5. **Обновляются в реальном времени** - когда ты редактируешь

Хорошие связи = хороший граф.

Не бойся добавлять много связей - Obsidian справится!
